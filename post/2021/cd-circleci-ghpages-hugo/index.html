Konfigurasi Hugo + Github Pages + CircleCI

Sabtu, 27 Februari 2021
hugo server

Hugo adalah Static Site Generator (SSG) yang dibuat dengan bahasa pemrograman Go-lang sehingga Hugo bisa menjadi SSG dengan kecepatan build yang super cepat, karena dalam aspek SSG kecepatan build adalah hal yang sangat penting untuk dipertimbangkan mencakup setiap dokumen, artikel, atau beberapa jenis konten lain akan di build satu-per-satu sehingga menjadi bentuk website penuh yang utuh.

Untuk mengunggah projek Hugo juga cukup mudah, hanya dengan melakukan proses build di local komputer kita lalu kemudian mengunggah hasil build tersebut ke hosting dan selesai, karena memang hasil build adalah file statik jadi kita bisa mengunggahnya di mana saja selama ada web server, dan tentunya Github Pages.

Github Pages adalah layanan hosting situs statik yang disediakan oleh Github untuk bisa membuat website dari file2 statik, fitur ini umumnya digunakan untuk membuat sebuah web dokumntasi maupun personal blog, namun tidak seperti hosting biasa, untuk menggunggah file pada Github Pages cukup dengan melakukan push ke repository yang dituju, jadi untuk sobat yang masih belum mengetahui tentang Github ini bisa mempelajari dasar-dasar dari VCS (Version Control System ya bukan yang anu).

Saya asumsikan sobat sudah bisa dan mempunyai projek Hugo ini, karena disini saya akan fokus dengan cara membuat automatisasi untuk proses penunggahan situs Hugo yang telah di push ke repository sehingga tidak perlu lagi melakukan proses build manual di local komputer sebelum deploy.

Pertama pada folder root projek Hugo, buat folder .circleci dan file config.yml di dalam-nya.

Lalu edit file config.yml seperti berikut:

version: 2.0

jobs:
    build:
        docker:
            - image: felicianotech/docker-hugo:latest
        working_directory: ~/<SITE_NAME>.github.io
        steps:
            - add_ssh_keys:
                fingerprints:
                    - "<USER_KEY_FINGERPRINT>"
            - checkout
            - run:
                name: Get current site
                working_directory: ~/
                command: git clone -b master git@github.com:<USERNAME/ORGANIZATION>/<SITE_NAME>.github.io.git public
            - run:
                name: Generate site
                working_directory: ~/<SITE_NAME>.github.io
                command: HUGO_ENV=production hugo -d ~/public
            - deploy:
                name: Deploy to Github Pages
                working_directory: ~/public
                command: |
                    git config credential.helper 'cache --timeout=120'
                    git config user.email "<YOUR_EMAIL>"
                    git config user.name "Deployment Bot"
                    git add .
                    git commit --allow-empty -m "Trigger deployment"
                    git push -q git@github.com:<USERNAME/ORGANIZATION>/<SITE_NAME>.github.io.git master                    

workflows:
  version: 2
  main:
    jobs:
    - build:
        filters:
          branches:
            only: source
